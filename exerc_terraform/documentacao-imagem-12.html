1. Edição de Código

Utilizamos o editor Nano diretamente no terminal para abrir o arquivo main.tf e adicionar uma nova tag ao recurso de Security Group. Esta é a forma comum de evoluir a infraestrutura: alterar o código e deixar o Terraform calcular a diferença.

2. O Conceito de Update In-Place

O símbolo de til (~) em amarelo indica uma mudança do tipo "updated in-place". Isso é uma excelente notícia em DevOps: significa que o Terraform consegue aplicar a alteração sem deletar e recriar o recurso, evitando qualquer queda (downtime) no serviço.

3. Adição de Metadados

O sinal de mais (+) em verde dentro do bloco de tags mostra exatamente o que está sendo adicionado: a chave Owner com o valor aluno-lab. As outras 3 tags que já existiam permanecem inalteradas (hidden).

4. Segurança Operacional

Ao rodar o plan antes do apply, confirmamos que apenas 1 mudança será feita e 0 recursos serão destruídos. Esse passo é fundamental para garantir que uma simples edição de tag não cause a destruição acidental de um banco de dados ou servidor.
