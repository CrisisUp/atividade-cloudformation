1. Aplicando o Update In-Place

Após validarmos o plano no passo anterior, executamos o terraform apply. Note que o Terraform reafirma o plano antes de agir: 1 to change. Ao digitarmos yes, a mudança de tag é enviada para a API da AWS.

2. Velocidade da Modificação

Como se tratava apenas de uma tag (metadado), o terminal exibe "Modifications complete after 0s". Isso demonstra a eficiência do Terraform em gerenciar apenas o que foi alterado, sem precisar reavaliar toda a infraestrutura física.

3. Resumo da Transação

O contador de recursos mostra 1 changed. É importante notar que 0 added e 0 destroyed, confirmando que a nossa infraestrutura original (o Security Group e a EC2 associada a ele) permanece intacta e funcional.

4. Persistência do ID

O sg_id exibido no output é idêntico ao anterior (sg-0147...). Isso prova que o recurso é o mesmo e que o Terraform apenas "carimbou" a nova informação nele, mantendo a integridade dos relacionamentos entre recursos.
