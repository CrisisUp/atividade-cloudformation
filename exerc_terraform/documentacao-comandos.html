Dicionário de Comandos

Entendendo cada passo da automação com Terraform

Instalação

Preparando o Gerenciador de Pacotes

sudo yum install -y yum-utils

Instala ferramentas auxiliares para o YUM (gerenciador de pacotes do Amazon Linux). O "utils" permite que o sistema manipule repositórios externos com mais facilidade.

Impacto: Prepara o terreno para adicionar o repositório oficial da HashiCorp.
Instalação

Adicionando Repositório HashiCorp

sudo yum-config-manager --add-repo https://rpm.releases.hashicorp.com/AmazonLinux/hashicorp.repo

Informa ao sistema operacional onde ele deve buscar os arquivos de instalação do Terraform. Sem isso, o comando 'install terraform' falharia por não encontrar o pacote nos repositórios padrão da Amazon.

Instalação

Instalando o Binário

sudo yum install -y terraform

Baixa e instala o executável do Terraform. A flag -y responde "sim" automaticamente para todas as confirmações de download.

Fluxo de Trabalho

Inicialização do Projeto

terraform init

O primeiro comando em qualquer projeto. Ele lê seus arquivos .tf, identifica que você usará a AWS e baixa os plugins (providers) necessários para a pasta oculta .terraform/.

Impacto: Cria o arquivo de trava (lock file) para garantir que todos usem a mesma versão dos plugins.
Fluxo de Trabalho

Validação de Sintaxe

terraform validate

Verifica se o seu código HCL possui erros de escrita, parênteses abertos ou referências a variáveis que não existem. É o "corretor ortográfico" do seu código.

Fluxo de Trabalho

Visualização do Plano

terraform plan

O comando mais importante para segurança. Compara o seu código com a infraestrutura real e gera um relatório: o que será criado (+), alterado (~) ou destruído (-).

Fluxo de Trabalho

Aplicação das Mudanças

terraform apply

Executa as ações planejadas. Ele solicita uma confirmação final ("yes") antes de realmente criar os recursos na nuvem.

Impacto: Altera a infraestrutura real e gera/atualiza o arquivo terraform.tfstate.
Gestão de Estado

Listagem de Recursos Gerenciados

terraform state list

Mostra todos os recursos que o Terraform "sabe" que existem. Se um recurso está na AWS mas não aparece aqui, o Terraform não o gerencia.

Gestão de Estado

Remoção de item do Estado

terraform state rm aws_security_group.lab

Faz o Terraform "esquecer" um recurso. O recurso NÃO é deletado da AWS, ele apenas para de ser rastreado pelo código.

Perigo: Se você rodar 'apply' depois disso, o Terraform tentará criar o recurso de novo, gerando conflitos.
Gestão de Estado

Importação de Recurso Existente

terraform import aws_security_group.lab sg-0147decfcf43db838

O oposto do 'rm'. Traz um recurso que já existe na AWS para dentro da gestão do seu código Terraform.

Avançado

Console Interativo

terraform console

Abre um terminal interativo para testar funções e interpolações de strings sem precisar criar recursos. Muito útil para testar lógicas complexas.

Ambientes

Uso de Arquivos de Variáveis

terraform plan -var-file="lab.tfvars"

Permite trocar todos os parâmetros (como tipo de instância e ambiente) apenas apontando para um arquivo diferente. Essencial para separar Lab de Produção.

Encerramento

Destruição da Infraestrutura

terraform destroy

Lê o arquivo de estado e remove TUDO que foi criado pelo projeto. É o comando de limpeza final para evitar custos desnecessários.

Impacto: Limpa o ambiente AWS e deixa o arquivo de estado vazio.
