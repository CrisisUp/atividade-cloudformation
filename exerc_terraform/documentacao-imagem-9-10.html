1. Confirmação de Segurança

O comando terraform apply não executa as mudanças imediatamente. Ele apresenta o plano uma última vez e solicita a confirmação manual do usuário. Somente após digitar exatamente yes é que o Terraform inicia as chamadas de API para a AWS.

2. Provisionamento em Tempo Real

Durante a execução, o Terraform fornece um log de progresso. Na imagem, observamos que a criação da instância EC2 levou exatamente 22 segundos. Esse tempo inclui a alocação do hardware e o início do processo de boot na infraestrutura da AWS.

3. Sincronização do Estado

A mensagem "Apply complete!" confirma que a infraestrutura real agora condiz 100% com o código. Nesse momento, o arquivo terraform.tfstate foi atualizado com o novo ID da instância (i-0445542e60c5bd1c3).

4. Visualização dos Resultados (Outputs)

Como definimos o arquivo outputs.tf anteriormente, o Terraform finaliza exibindo os valores solicitados. Agora temos o ID real da máquina e do Security Group disponíveis no terminal para uso em scripts ou simples conferência.
