Documentação Técnica Terraform

Guia detalhado de infraestrutura como código (IaC) para o projeto TechStock

Setup

1. Instalação do Terraform

1sudo yum install -y yum-utils 2sudo yum-config-manager --add-repo \ 3 https://rpm.releases.hashicorp.com/AmazonLinux/hashicorp.repo 4sudo yum install -y terraform 5terraform version

Comandos de Instalação (Amazon Linux)

Linha 1: Instala o pacote yum-utils, que fornece ferramentas para gerenciar repositórios e pacotes adicionais.
Linhas 2-3: Adiciona o repositório oficial da HashiCorp ao gerenciador de pacotes YUM. Isso garante que estamos baixando a versão oficial e assinada.
Linha 4: Instala efetivamente o binário do Terraform no sistema de forma não interativa (-y).
Linha 5: Verifica se a instalação foi bem-sucedida imprimindo a versão do Terraform instalada.
Configuração

2. provider.tf

1terraform { 2 required_version = ">= 1.6.0" 3 required_providers { 4 aws = { 5 source = "hashicorp/aws" 6 version = "~> 5.0" 7 } 8 } 9} 10 11provider "aws" { 12 region = "us-east-1" 13}

Bloco de Configuração do Terraform

Linha 1: O bloco terraform contém as configurações globais do Terraform para o projeto.
Linha 2: Define a versão mínima do binário do Terraform exigida para executar este código (evita incompatibilidades).
Linhas 3-8: O bloco required_providers especifica quais plugins (providers) são necessários. Definimos o provider aws da HashiCorp com a versão ~> 5.0 (permitindo apenas atualizações de patch e minor dentro da v5).
Linhas 11-13: O bloco provider "aws" configura o plugin da AWS. Definimos a região us-east-1. No CloudShell, as credenciais são injetadas automaticamente via LabRole.
Recursos

3. main.tf (Data Sources & Security Group)

1data "aws_ami" "al2023" { 2 most_recent = true 3 owners = ["amazon"] 4 filter { 5 name = "name" 6 values = ["al2023-ami-2023*-x86_64"] 7 } 8} 9 10resource "aws_security_group" "lab" { 11 name = "techstock-tf-lab-sg" 12 description = "SG para o lab de Terraform" 13 14 ingress { 15 from_port = 80 16 to_port = 80 17 protocol = "tcp" 18 cidr_blocks = ["0.0.0.0/0"] 19 } 20 21 egress { 22 from_port = 0 23 to_port = 0 24 protocol = "-1" 25 cidr_blocks = ["0.0.0.0/0"] 26 } 27}

Infraestrutura Base

Linhas 1-8: O data source busca informações da AWS sem criar nada. Aqui, localizamos a AMI do Amazon Linux 2023 mais recente, filtrando por nome e proprietário (Amazon).
Linha 10: Define o recurso de Security Group chamado "lab" internamente no Terraform.
Linhas 14-19: Bloco de entrada (ingress). Permite tráfego na porta 80 (HTTP) vindo de qualquer endereço IP (0.0.0.0/0).
Linhas 21-26: Bloco de saída (egress). Permite todo o tráfego de saída (protocolo -1) para qualquer destino.
Compute

4. main.tf (EC2 Instance)

1resource "aws_instance" "lab" { 2 ami = data.aws_ami.al2023.id 3 instance_type = var.instance_type 4 5 vpc_security_group_ids = [aws_security_group.lab.id] 6 iam_instance_profile = "LabInstanceProfile" 7 8 tags = { 9 Name = "techstock-tf-lab-ec2" 10 Environment = var.environment 11 } 12}

Instância EC2 com Referências Dinâmicas

Linha 1: Declara uma instância EC2.
Linha 2: Referência dinâmica ao ID da AMI encontrado pelo data source anterior. Se a AMI mudar na AWS, o Terraform detectará.
Linha 3: Usa o valor de uma variável definida no arquivo variables.tf.
Linha 5: Referência cruzada. O Terraform obtém o ID do Security Group criado anteriormente, estabelecendo uma dependência implícita (cria o SG antes da EC2).
Linha 6: Atribui o perfil IAM necessário no ambiente Learner Lab.
Linhas 8-11: Aplica tags organizacionais, interpolando a variável environment.
Variáveis

5. variables.tf

1variable "instance_type" { 2 type = string 3 default = "t2.micro" 4 description = "Tipo da instancia EC2" 5 validation { 6 condition = contains(["t2.micro", "t2.small"], var.instance_type) 7 error_message = "Use t2.micro ou t2.small no Learner Lab." 8 } 9} 10 11variable "environment" { 12 type = string 13 default = "lab" 14}

Declaração e Validação

Linhas 1-4: Define a variável instance_type como string com um valor padrão.
Linhas 5-8: Bloco de validação customizada. A função contains verifica se o valor fornecido está na lista permitida. Se falhar, exibe a error_message.
Linhas 11-14: Variável simples para definir o nome do ambiente, facilitando a reutilização do código para Dev, Lab ou Prod.
Outputs

6. outputs.tf

1output "instance_id" { 2 value = aws_instance.lab.id 3} 4 5output "public_ip" { 6 value = aws_instance.lab.public_ip 7}

Valores de Saída

Linhas 1-3: Exporta o ID da instância criada. Útil para scripts de automação ou para consulta rápida no terminal após o apply.
Linhas 5-7: Mostra o IP público da instância, permitindo o acesso via SSH ou navegador sem precisar ir ao console AWS.
Avançado

7. sgs.tf (Uso de for_each)

1variable "app_ports" { 2 type = map(number) 3 default = { 4 backend = 3000 5 monitoring = 9090 6 } 7} 8 9resource "aws_security_group" "app_sgs" { 10 for_each = var.app_ports 11 12 name = "techstock-${each.key}-sg" 13 14 ingress { 15 from_port = each.value 16 to_port = each.value 17 protocol = "tcp" 18 cidr_blocks = ["0.0.0.0/0"] 19 } 20}

Iteração com Maps

Linhas 1-7: Define um mapa onde a chave é o nome do serviço e o valor é a porta.
Linha 10: O for_each diz ao Terraform para criar um recurso para cada item no mapa app_ports.
Linha 12: each.key refere-se à chave do mapa (ex: "backend"). Usamos interpolação de string para gerar nomes únicos.
Linhas 15-16: each.value refere-se ao valor do mapa (ex: 3000). Isso permite configurar portas diferentes para cada SG em um único bloco de código.
Avançado

8. Variáveis Sensíveis e Locals

1variable "db_password" { 2 type = string 3 sensitive = true 4} 5 6locals { 7 app_name = "techstock" 8 full_name = "${local.app_name}-${var.environment}" 9}

Segurança e Reutilização

Linha 3: sensitive = true impede que o valor da variável seja exibido no terminal (stdout) durante o plan ou apply. Essencial para senhas e tokens.
Linhas 6-9: O bloco locals define variáveis internas calculadas.
Linha 8: Demonstra interpolação de string combinando um valor local (app_name) com uma variável de entrada (environment) para criar um nome composto reutilizável.
