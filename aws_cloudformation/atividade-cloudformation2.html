Primeira Stack — O Fluxo de Criação
Console
ℹ️ Em 2026, o Composer está integrado diretamente no fluxo de preparação do modelo do CloudFormation.
💡 O que é uma Stack?
Uma stack é um conjunto de recursos AWS gerenciados como uma unidade. Ao deletar a stack, o CloudFormation garante que tudo seja limpo na ordem correta.

Passo 1 — Inicie a criação no Console

  1. AWS Console → CloudFormation → Clique em Criar pilha → Com novos recursos (padrão)
  2. Em "Pré-requisito", escolha Construir a partir do Infrastructure Composer
  3. Clique em Abrir no Infrastructure Composer e cole o YAML na aba Template:
AWSTemplateFormatVersion: '2010-09-09'
Description: Lab 2026 - EC2 moderna (Amazon Linux 2025)

Parameters:
  InstanceType:
    Type: String
    Default: t3.micro
    AllowedValues: [t3.micro, t3.small]
    Description: Tipo da instancia

Resources:
  LabEC2:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: ami-0c101f26f147fa7fd 
      InstanceType: !Ref InstanceType
      IamInstanceProfile: LabInstanceProfile
      Tags:
        - Key: Name
          Value: !Sub "${AWS::StackName}-modern-ec2"

Outputs:
  InstanceId:
    Value: !Ref LabEC2
  PrivateIP:
    Value: !GetAtt LabEC2.PrivateIp

📸 [PDF - Seção 1: Print 1] Clique em Salvar e depois em Continuar no CloudFormation.

Parameters & Mappings — Infra Parametrizada
Dynamic
ℹ️ Mappings permitem que o mesmo template funcione em qualquer região sem alterações manuais. 
AWSTemplateFormatVersion: '2010-09-09'
Description: Template Parametrizado - Padrões 2026

Parameters:
  Environment:
    Type: String
    Default: dev
    AllowedValues: [dev, prod]

Mappings:
  RegionalSettings:
    us-east-1: 
      AMI: ami-0c101f26f147fa7fd
      Size: t3.micro
    us-west-2: 
      AMI: ami-07355fe79b493752d
      Size: t3.micro

Resources:
  AppServer:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: !FindInMap [RegionalSettings, !Ref "AWS::Region", Size]
      ImageId: !FindInMap [RegionalSettings, !Ref "AWS::Region", AMI]
      IamInstanceProfile: LabInstanceProfile
      Tags:
        - Key: Env
          Value: !Ref Environment

Prática

Crie uma stack chamada techstock-parametrizado. Experimente mudar o parâmetro para ver os Mappings em ação.

Outputs Cross-Stack e Dependências
Modulação
ℹ️ Use `Export` e `!ImportValue` para compartilhar recursos entre diferentes stacks.

Parte A — Infra (Export)

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  SGApp2026:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: SG Modular
      SecurityGroupIngress:
        - IpProtocol: tcp
          FromPort: 80
          ToPort: 80
          CidrIp: 0.0.0.0/0
Outputs:
  SharedSGId:
    Value: !Ref SGApp2026
    Export:
      Name: !Sub "${AWS::StackName}-SharedSG"

Nome: techstock-infra-lab

Parte B — App (Import)

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  AppInstance:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: ami-0c101f26f147fa7fd
      InstanceType: t3.micro
      SecurityGroupIds:
        - !ImportValue "techstock-infra-lab-SharedSG"

Nome: techstock-app-lab
